# Punk Cafe

Sick of Instagram so I made my own.
- 🤫 Invite-only
- 🤗 Everyone follows everyone else
- 🥳 Chronological feed
- 😎 98.6% code coverage from Node's built-in test runner

## The Stack
This is the back-end: TypeScript and Prisma with PostgreSQL.

Running on a VPS with Caddy for the reverse proxy. Kept alive with a SystemD service. Docker containerizes the database.

I have a self-hosted instance of [Uptime Kuma](https://uptimekuma.org/) to monitor uptime. It's integrated with [ntfy](https://ntfy.sh/) to ping my phone in case something goes wrong! 

The front-end is an iOS app written in Swift.

## Architecture

To keep related concerns together, I went for an [entity-based](https://github.com/Underwaterr/Punk-Cafe-API/tree/main/server/entities) folder structure. Each entity gets their own model, controller, router, schema and test file. 

Authentication is handled with token-based sessions issued on login and checked with each request. No cookies since this is iOS-only. Registration requires invitation codes generated by existing members.

Image uploads (the heart of the app) are validated and compressed server-side, using magic-bytes to determine file type. Image processing is handled via [Sharp](https://sharp.pixelplumbing.com/). 

Prisma handles database schemas and relations, including data integrity, indexing and cascading deletes.
